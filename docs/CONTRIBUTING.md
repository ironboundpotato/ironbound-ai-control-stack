Contributing to TE-Mobile
System: Ironbound AI — TE-Mobile Engine
Author: Kevin Gilbert
TE-Mobile is a controlled architecture.
Contributions focus on clarity, examples, tests, and documentation improvements.
1. Operator Principles
All contributors must follow core Ironbound rules:
No drift (tone, density, scope, intent).
No implicit inference — all changes must be explicit.
Role separation — Architect / Editor / Stress Tester / Polisher remain distinct.
Density stability — avoid compression artifacts or unstable expansion.
Human primacy — no autonomous chains without operator instruction.
2. Contribution Workflow
Fork the repository
Create a feature branch
Follow ILP v2.1:
Architect → Editor → Stress Tester → Polisher
Reference related docs in /docs
Open a Pull Request
3. Documentation Requirements
All new documentation must include:
Title
Purpose
Version or date
Clear section structure
Mapping to INDEX.md
4. Non-Acceptable Contributions
Drifted or uncontrolled model output
Breaking kernel laws or constraints
Files contradicting whitepaper or architecture
Expansion of system scope without operator approval
Changes to LICENSE or Whitepaper
End of CONTRIBUTING Guidelines
